# Community Design Constraints

These constraints define how the BuzzPi community operates. They ensure the project remains welcoming, transparent, and sustainable.

- All communication in project spaces follows the Code of Conduct. Violations are addressed promptly and privately.
- Major technical decisions follow the RFC process. There is no shortcut around community review for significant changes.
- RFCs are public and remain in the repository permanently, regardless of outcome. Every decision (and rejection) is documented.
- Maintainers are selected by demonstrated contribution quality, not by employer, affiliation, or seniority.
- There are no private decision channels. All governance discussions happen in public.
- The project does not accept corporate sponsorship that compromises the open specification or user privacy.
- Plugin certification is open to anyone. There are no paid tiers for certification.
- Documentation is maintained alongside code. A PR that adds a feature without documentation is incomplete.
- The project releases on a predictable cadence. Users and downstream integrators should not be surprised by release timing.
